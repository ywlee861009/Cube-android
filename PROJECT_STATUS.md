# 프로젝트 진행 상황 요약 (2026-04-15)

오늘 진행된 주요 작업 내용과 현재 프로젝트의 상태를 정리합니다.

## 🟢 완료된 작업

### 1. 큐브 회전 논리 오류 수정
- **문제**: `U`(위), `D`(아래), `E`(중간 수평층) 회전 시 인접한 면들의 색상이 시계 방향(CW)이 아닌 반대로 순환되는 문제 발견.
- **해결**: `logic.js`의 `MOVES` 정의에서 `cycle4` 호출 순서를 표준 시계 방향에 맞게 수정 완료.
- **결과**: 모든 18개 기본 무브 및 중간 레이어 회전이 논리적으로 정확하게 동작함.

### 2. Edge-to-Edge 및 시스템 바 패딩 대응
- **문제**: WebView가 상태바와 네비게이션 바 영역을 덮으면서 UI 요소(Moves 텍스트, 버튼)가 겹치는 현상 발생.
- **해결**: 
    - `MainActivity.kt`에 `enableEdgeToEdge()` 적용.
    - `WindowInsets`를 통해 시스템 바 높이를 계산하여 WebView(JavaScript)로 전달하는 브릿지(`setInsets`) 구축.
    - `cube.html`에서 CSS 변수(`--safe-top`, `--safe-bottom` 등)를 사용하여 UI 여백 자동 조정.
    - `viewport-fit=cover` 설정으로 전체 화면 대응 강화.
- **안정성**: 페이지 로딩 완료(`onPageFinished`) 후에도 인셋을 다시 전달하도록 하여 통신 누락 방지.

### 3. 빌드 오류 수정 및 안정화
- **문제**: `MainActivity.kt`에서 `backgroundColor` 프로퍼티 참조 오류로 인한 빌드 실패.
- **해결**: `setBackgroundColor(Color.TRANSPARENT)` 메서드 호출 방식으로 수정하여 빌드 성공 확인.
